===============================================================================
NAVIGATION — CD I
===============================================================================

OVERVIEW
-------------------------------------------------------------------------------

The:

    cd

command stands for:

    Change Directory

It is used to move from one directory to another directory in the filesystem.

Think of `cd` as the command-line equivalent of opening a folder in:

    Windows File Explorer

or:

    macOS Finder


MEMORY:

    cd

      ↓

    Change Directory

      ↓

    "Move me into another directory."


===============================================================================
WHAT DOES CD DO?
===============================================================================

The:

    cd

command changes the:

    working directory


Remember:

The working directory is the directory you are currently inside.


Example:

Current working directory:

    blog/


Filesystem:

    blog/
    |
    +-- 2014/
    |
    +-- 2015/
    |
    +-- hardware.txt


To move into:

    2015/


use:

    cd 2015


Result:

    blog/
      |
      +-- 2015/
            ^
            |
        YOU ARE HERE


===============================================================================
BASIC CD SYNTAX
===============================================================================

Syntax:

    cd directory_name


Example:

    cd 2015


When shown with the shell prompt:

    $ cd 2015


Breakdown:

    cd

        command


    2015

        argument


===============================================================================
WHAT IS AN ARGUMENT?
===============================================================================

When a:

    file

    directory

or:

    program

is passed into a command, it is called an:

    argument


Example:

    cd 2015


Breakdown:

    cd

        COMMAND


    2015

        ARGUMENT


Visual:

    cd       2015
    |          |
    |          |
    v          v
  COMMAND   ARGUMENT


The argument tells the command what to act upon.


===============================================================================
COMMAND + ARGUMENT
===============================================================================

Many command-line commands follow this general pattern:

    command argument


Example:

    cd 2015


Meaning:

    COMMAND:

        cd

        Change directory


    ARGUMENT:

        2015

        Directory to enter


Mental Model:

    COMMAND

        +

    ARGUMENT

        =

    SPECIFIC ACTION


Example:

    cd 2015

        ↓

    Change directory to 2015/

CHANGING INTO A CHILD DIRECTORY
===============================================================================

Suppose your current working directory is:

    /home/ccuser/workspace/blog


Filesystem:

    blog/
    |
    +-- 2014/
    |
    +-- 2015/
    |
    +-- hardware.txt


Run:

    cd 2015


The working directory changes to:

    /home/ccuser/workspace/blog/2015


Visual:

BEFORE:

    blog/
      ^
      |
  YOU ARE HERE
      |
      +-- 2014/
      |
      +-- 2015/


COMMAND:

    cd 2015


AFTER:

    blog/
      |
      +-- 2015/
            ^
            |
        YOU ARE HERE


===============================================================================
VERIFYING THE DIRECTORY CHANGE
===============================================================================

Before changing directories:

    $ pwd


Output:

    /home/ccuser/workspace/blog


Change directory:

    $ cd 2015


Check location again:

    $ pwd


Output:

    /home/ccuser/workspace/blog/2015


Workflow:

    pwd

      ↓

    /home/ccuser/workspace/blog


    cd 2015

      ↓

    MOVE INTO 2015/


    pwd

      ↓

    /home/ccuser/workspace/blog/2015


===============================================================================
MOVING UP ONE DIRECTORY
===============================================================================

It is also important to move from a child directory back to its parent
directory.

Use:

    cd ..


The:

    ..

argument represents:

    the parent directory

or:

    the directory one level above the current directory


===============================================================================
UNDERSTANDING ..
===============================================================================

The symbol:

    ..

means:

    Parent Directory


Example filesystem:

    blog/
    |
    +-- 2015/
          |
          +-- article.txt


If your working directory is:

    blog/2015/


then:

    ..

refers to:

    blog/


Visual:

    blog/               ← ..
      |
      +-- 2015/         ← current directory
            ^
            |
        YOU ARE HERE


Command:

    cd ..


Result:

    blog/
      ^
      |
  YOU ARE HERE


===============================================================================
CD .. EXAMPLE
===============================================================================

Assume the current working directory is:

    /home/ccuser/workspace/blog/2015


Check location:

    $ pwd


Output:

    /home/ccuser/workspace/blog/2015


Move up one directory:

    $ cd ..


Check location again:

    $ pwd


Output:

    /home/ccuser/workspace/blog


The command:

    cd ..


moved the working directory from:

    /home/ccuser/workspace/blog/2015


to:

    /home/ccuser/workspace/blog

===============================================================================
COMMON BEGINNER MISTAKES
===============================================================================

MISTAKE 1:

Trying to `cd` into a file.


Example:

    cd hardware.txt


Problem:

    hardware.txt

is a file, not a directory.


The `cd` command changes into:

    directories


not regular files.


-------------------------------------------------------------------------------
MISTAKE 2:

Using a directory that does not exist.


Example:

    cd 2020


If:

    2020/

does not exist in the specified location, the shell will return an error.


-------------------------------------------------------------------------------
MISTAKE 3:

Forgetting where you are.


Solution:

    pwd


Then inspect available directories:

    ls


Then navigate:

    cd directory_name

===============================================================================
KEY TAKEAWAYS
===============================================================================

- `cd` stands for:

      Change Directory

- The `cd` command changes the current working directory.

- The directory name passed to `cd` is an:

      argument

- Example:

      cd 2015

  moves into the `2015/` directory.

- The symbol:

      ..

  represents the parent directory.

- The command:

      cd ..

  moves up one directory level.

- `pwd`, `ls`, and `cd` form the foundation of command-line filesystem
  navigation.

- `pwd` shows your current location.

- `ls` shows the contents of the current directory.

- `cd` changes your current location.
